2、讲讲promise，promise的3种状态和状态转换？

Promise中的回调函数是异步执行的，也就是说，它们会在当前同步代码块执行完成后才会执行。这样可以避免阻塞主线程，提高程序的响应速度。

then()方法的链式调用也是异步的，每个then()方法中的回调函数都会在前一个then()方法中的回调函数执行完成后才会执行。这样可以保证then()方法中的回调函数按照指定的顺序依次执行。

3、说下call、apply、bind方法的作用和区别？

作用都是改变函数的执行上下文

  • call()方法接收的是一个参数列表，依次传递给函数。
  • apply()方法接收的是一个参数数组，作为函数的参数。
  • bind()方法不会立即执行函数，而是返回一个绑定了执行上下文的新函数，你可以在之后再调用新函数。

5、实现防抖方法？

防抖和节流都是用来控制函数调用频率的技术。

防抖指的是在事件被触发后，一定时间内再次触发时不会执行函数，只有在这段时间结束后才会再次执行函数。这样可以防止一个函数被触发多次，从而减少不必要的计算。

节流指的是在一定时间内只允许一次函数调用，当函数被触发时，如果在这段时间内还有其他函数调用，那么这些调用将被忽略，只有当时间结束后才会执行函数。这样可以限制函数的调用频率，从而避免对系统造成过度负荷。

通俗来说，防抖就是在一段时间内只执行一次函数，而节流就是限制函数的调用频率。

6、实现深拷贝和浅拷贝？

在 JavaScript 中，可以使用 JSON.parse() 和 JSON.stringify() 来实现深拷贝。首先，将要拷贝的对象使用 JSON.stringify() 转换为字符串，然后再使用 JSON.parse() 方法将字符串转换回对象。这样，就可以得到原对象的一个深拷贝。

8、什么是柯里化？

柯里化（Currying）是一种函数式编程的概念，它指的是将原来接受两个参数的函数转换为接受一个参数的函数，然后返回一个接受剩余参数的函数的过程。这个新的函数接受一个参数，并返回一个接受剩余参数的函数。这样通过多次调用，就可以达到传递所有参数的目的。

9、如何理解原型与原型链？

原型链的作用是让对象能够继承属性和方法，也就是说，一个对象可以从它的原型对象继承属性和方法，而它的原型对象又可以从它的原型对象继承属性和方法，以此类推。例如，Object.prototype 上定义的 toString() 方法可以被所有对象继承。

10、js有哪些继承方法？

  • 原型链继承：原型链继承是通过将一个对象的原型指向另一个对象来实现继承。
  • 构造函数继承：构造函数继承是通过在子类构造函数中调用父类构造函数来实现继承。
  • 实例继承：实例继承是通过将父类实例作为子类实例的原型来实现继承。
  • 拷贝继承：拷贝继承是通过将父类的属性和方法拷贝到子类中来实现继承。

在实际开发中，可以根据需要选择合适的继承方式。例如，如果想要在子类中重写父类的方法，可以使用构造函数继承或实例继承；如果想要保证父类的属性和方法不被更改，可以使用拷贝继承。

15、vue父子组件挂载顺序是什么？

加载渲染过程

父 beforeCreate -> 父 created -> 父 beforeMount -> 子 beforeCreate -> 子 created ->子 beforeMount -> 子 mounted -> 父 mounted

更新过程

父 beforeUpdate -> 子 beforeUpdate -> 子 updated -> 父 updated

销毁过程

父 beforeDestroy -> 子 beforeDestroy -> 子 destroyed -> 父 destroyed

16、讲一下vue组件的computed和watch的区别？

应用场景不同

computed用在根据data属性或者其他computed计算得到一个新值的情况，computed的值一般被用在渲染中。

watch用在监听数据变化，然后做一些有副作用的操作的场景。

执行过程不同

在依赖的data属性变化后，computed并不会重新计算新的值，而是等到访问的时候再判断，如果依赖的data有改动则重新计算并返回结果，如果依赖的data没有改动，就不计算，直接返回当前结果。

依赖的数据变化后就会执行watch的回调。

17、路由有哪两种模式？默认是哪种模式？两种模式区别是什么？讲一下vue-router原理？

前端路由有两种模式，HTML5和hash，这两种模式本质是不同的底层浏览器技术，但是上层Vue Router做了统一化的封装，因此在我们开发组件和配置路由时候使用这两种模式的区别并不大。默认是hash模式。

这两种模式有几个主要区别

\1. HTML5模式的路由没有”#”字符，而是在域名后直接写路径，更加优雅

\2. 由于#后面的字符不会发给服务器，因此hash路由SEO比较差

\3. HTML5需要服务器在访问不同的路径时候都能fallback到index.html，因此相对麻烦

前端路由的原理关键有2点

\1. 可以修改url，但不会引起刷新，从而在不刷新的页面的情况下跳转路由。

\2. 监听url改变，根据url渲染对应组件。

当用户在浏览器中访问 URL 时，Vue Router 会查找与当前 URL 匹配的路由规则，并显示对应的组件。如果没有匹配的路由规则，则显示一个 404 页面。
